# מערכת הכנה למבחן — נושאים מתקדמים בתכנות

מערכת לימוד מקומית, אינטראקטיבית ופרטית להכנה למבחן בקורס BGU **"Advanced Topics in Programming"** (מרצה: ד"ר דודי בן שמעון). כל התוכן נוצר מניתוח בפועל של מצגות ההרצאה, מצגות התרגול והקוד המצורף, ו-12 מבחני עבר משנים 2018–2025 (כולל מסמכים סרוקים שעברו תמלול ידני). המערכת פועלת **לגמרי אופליין** — אין שרת חיצוני, אין API בתשלום, וההתקדמות נשמרת רק במחשב שלך (localStorage).

## מה יש במערכת

- **16 נושאי לימוד** מ-Java platform/JVM ועד Blockchain, כל אחד ביחידת לימוד תלת-שכבתית (אינטואיציה → ידע למבחן → יישום), כולל שאלות בדיקה מובנות בתוך כל יחידה.
- **293 שאלות אמריקאיות פעילות** במאגר הליבה המאומת (מתוך 294 במאגר, ~74% מקוריות בסגנון מבחני עבר, ~25% מאומתות ממבחני עבר אמיתיים).
- **28 שאלות נוספות** בשלושה מאגרים משלימים (מסך "שאלות נוספות" נפרד), מחולצות ומאומתות ממקורות תרגול נוספים - מסומנות בבירור כנפרדות מהמאגר הליבה, ניתן לתרגל אותן ישירות בתוך "למידה" (בתחתית כל נושא) או במסך הייעודי, ואפשר לכלול אותן גם ב-Quiz Me וגם במבחן מדומה כתוספת אופציונלית.
- **188 כרטיסיות זיכרון** עם חזרה מרווחת (Leitner).
- **7 מבחנים מדומים** בני 20 שאלות (100 נק'), כולל מבחן אחד שהוא **שחזור אמיתי של מבחן 2025** (20/20 מאומת).
- מבחן אבחון, תרגול אדפטיבי, מחברת טעויות אוטומטית, חזרה אחרונה (Last-Minute), חיפוש חופשי בכל התוכן (נושאים, חומר לימוד, שאלות וכרטיסיות, עם הדגשת התאמות), ותוכנית לימוד דינמית ל-3 ימים.
- **סדר תשובות יציב**: סדר האפשרויות (A-E) בכל שאלה נקבע פעם אחת לכל ניסיון ולא משתנה בעקבות רענון, שינוי רמת ביטחון, סימון "למדתי" או כל אינטראקציה אחרת בעמוד - הניקוד תמיד מבוסס על זהות התשובה, לא על מיקומה המוצג.
- ממשק בעברית מלאה עם תמיכת RTL בכל מסך, כאשר קוד, מונחים טכניים, ביטויים מעורבים (כולל סוגריים, מרכאות ואופרטורים) מוצגים תמיד ב-LTR ובסדר הנכון, גם בתוך משפט עברי ארוך ומורכב.

## דרישות מערכת

- **Node.js גרסה 18 ומעלה** (נבדק עם v24). ניתן להוריד מ-https://nodejs.org
- אין צורך ב-Python בזמן הרצת האתר (Python נדרש רק אם רוצים להריץ מחדש את סקריפטי החילוץ מהחומרים הגולמיים — ראו "פיתוח נוסף" למטה).

## התקנה והפעלה

### דרך מהירה (Windows)

1. הרץ פעמיים על `setup.bat` — יתקין את כל התלויות הנדרשות (עשוי לקחת מספר דקות בפעם הראשונה).
2. הרץ פעמיים על `start.bat` — יפעיל את השרת המקומי.
3. פתח דפדפן בכתובת שתוצג בחלון (בדרך כלל `http://localhost:5173`).
4. כדי לעצור — סגור את חלון ה-CMD או לחץ `Ctrl+C` בתוכו.

### דרך ידנית (כל מערכת הפעלה)

```bash
npm install
npm run dev
```

ואז פתחו את הכתובת שתוצג בטרמינל (בדרך כלל `http://localhost:5173`).

להרצת גרסת production מקומית:

```bash
npm run build
npm run preview
```

## שימוש בהתקדמות אישית

- **שמירה אוטומטית**: ההתקדמות (תשובות, ציונים, מחברת טעויות, כרטיסיות) נשמרת אוטומטית ב-localStorage של הדפדפן ונטענת מחדש בכל פתיחה.
- **גיבוי**: בסרגל הצד (בתחתית) לחצו על "גיבוי התקדמות" — יורד קובץ JSON עם כל הנתונים שלכם.
- **שחזור מגיבוי**: לחצו "שחזור מגיבוי" ובחרו את קובץ ה-JSON שגיביתם.
- **איפוס**: לחצו "איפוס התקדמות" — פעולה בלתי הפיכה (מומלץ לגבות קודם).

> שימו לב: ההתקדמות שמורה **בדפדפן הספציפי** בו השתמשתם. מעבר לדפדפן אחר או מחיקת נתוני האתר ימחקו את ההתקדמות המקומית — לכן מומלץ לגבות באופן קבוע, במיוחד לפני המבחן.

## הוספת שאלה חדשה

1. פתחו את `extracted_materials/clusters/` ובחרו קובץ cluster מתאים (או צרו קובץ חדש), או ערכו ישירות את `src/data/questions.json`.
2. כל שאלה חייבת להכיל: `id` ייחודי, `topicIds` (חייב להתאים לאחד מ-16 ה-ids ב-`src/data/topics.json`), `stemHe`, בדיוק 5 `options` (מזהים a–e, טקסט ייחודי), `correctOptionId` תקין, `explanation`, ו-`optionExplanations` לכל חמש האפשרויות.
3. אם ערכתם קובץ cluster, הריצו מחדש:
   ```bash
   python scripts/mergeContent.py
   npx tsx scripts/assembleMockExams.ts
   ```
4. הריצו validation (ראו למטה) לפני שסומכים על השאלה החדשה.

## הרצת בדיקות (validation, tests, lint)

```bash
npm run validate:data   # בודק את מאגר השאלות (ליבה + מאגרים נוספים): 5 אפשרויות, תשובה אחת נכונה, IDs ייחודיים ללא כפילויות, אין needsReview פעיל וכו'
npm run test            # מריץ את כל בדיקות היחידה (Vitest) - כולל בדיקות סדר-תשובות יציב, בידוד מצב-שאלה, ורינדור RTL/LTR
npm run lint            # בודק סגנון קוד (ESLint)
npm run typecheck       # בודק טיפוסי TypeScript
npm run build           # בונה גרסת production ומוודא שהכול תקין
npm run verify          # מריץ את כל הבדיקות שלמעלה ברצף אחד (typecheck+test+lint+validate:data+build)
```

## בעיות נפוצות ופתרונות

**"node אינו מזוהה כפקודה"** — Node.js לא מותקן או לא ב-PATH. התקינו מ-https://nodejs.org והפעילו מחדש את הטרמינל.

**`npm install` נכשל** — ודאו חיבור אינטרנט (נדרש רק להתקנה החד-פעמית של תלויות; השימוש היומיומי באתר לגמרי אופליין). נסו למחוק את תיקיית `node_modules` ואת `package-lock.json` ולהריץ שוב.

**האתר לא נטען / מסך לבן** — פתחו את כלי הפיתוח בדפדפן (F12) ובדקו שגיאות בלשונית Console. ודאו שהרצתם `npm run build` בהצלחה אם אתם משתמשים ב-`npm run preview`.

**איבדתי את ההתקדמות שלי** — אם לא גיביתם, אין דרך לשחזר נתונים שנמחקו מ-localStorage. מומלץ לגבות באופן קבוע דרך הכפתור בסרגל הצד.

**רוצה להריץ מחדש את חילוץ החומרים מהמקור** — דורש Python 3.10+ עם `pip install python-pptx python-docx PyMuPDF`, והרצת הסקריפטים בתיקיית `scripts/extract/` בסדר: `extract_pptx.py`, `extract_docx.py`, `extract_pdf_text.py`, `extract_zip.py`, `extract_csv.py`, `extract_pdf_render.py`, ולבסוף `build_manifest.py`. זהו תהליך מתקדם המיועד רק אם רוצים לעדכן את החומרים המקוריים.

## הגבלות ידועות

- חלק קטן מהשאלות (28 מתוך 293, כ-9.5%) מסומנות `confidence: "medium"` — מבוססות על החומר אך מרחיבות מעט מעבר לניסוח המילולי המדויק של השקופיות. עדיין עברו בדיקה, אך שווה לתת בהן אמון מעט נמוך יותר בהשוואה לשאר.
- משך המבחן האמיתי אינו ידוע בוודאות (מקורות שונים הראו 90 ו-150 דקות) — ניתן לקבוע זמן ידנית במסך המבחן המדומה.
- מספר שאלות זהות (או כמעט זהות) חוזרות בין שנות מבחן שונות (2018/2024) באופן מכוון — זהו תיעוד אמיתי של שאלות שחוזרות בפועל, לא כפילות שגויה.
